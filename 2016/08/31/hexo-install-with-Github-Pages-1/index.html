运用Github Pages和Hexo搭建静态博客

发表于 2016-08-31

一、前言
这是一篇使用Github Page和Hexo搭建静态博客的自我总结，在搭建过程中出现的问题也会进行补充。
我是一个计算机小白，CSDN和LOFTER等博客，不能说改就改，所以搭建一个属于自己的博客感觉很炫酷。
如果你初次接触，先看看下面一些文档，有助于你更好的操作，本教程只针对Windows用户。

  • Git
  • Github
  • Github Pages
  • Hexo
  • Markdown语法
    对于搭建教程和使用更多可以参考以下网站：
  • Hexo官方文档
  • 知乎Hexo话题

二、Git
2.1 安装Git
官方网站下载安装Git
2.2 创建一个仓库
在自己的GitHub账号下创建一个新的仓库，命名为username.github.io（username是你的账号名)。
2.3 配置Git
2.3.1 设置用户名及Email
安装完成后需要配置你的Git，就是设置你的用户名和Email，在任意位置右键选择Git Bash，键入

$ git config --global user.name "username"
$ git config --global user.email "username@example.com"

2.3.2 SSH配置
为了能在本地使用Git管理Github上面的项目，我们进行SSH key配置。
2.3.2.1检查电脑是否有SSH key

$ ls -al ~/.ssh

如果提示列出文件id_ras.pub和id_rsa，证明已存在SSH key。
2.3.2.2 若没有SSH key，生成新的

$ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

之后一路回车就可以了。
2.3.2.3 向ssh-agent添加key。
首先确保ssh-agent可运行：

$ ssh-agent -s

然后添加SSH key：

$ ssh-add ~/.ssh/id_rsa

2.3.2.4 向Github添加SSH key。
首先，拷贝key：

clip < ~/.ssh/id_rsa.pub

然后，在GitHub右上方点击头像，选择”Settings”，在右边的”SSH and GPG keys”,点击”New SSH key”按钮，添加上去。测试：

$ ssh -T git@github.com

一直回车，最后输入yes，出现这个就是成功了。

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

三、Hexo
3.1 安装Hexo
安装之前需要在电脑上安装好以下程序：

  • Git
  • Node.js

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。

$ npm install -g hexo-cli

3.2 建站
安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。

$ hexo init folder>
$ cd <folder>
$ npm install

这样，我们就已经搭建起本地的Hexo博客了。可以先在对应文件夹下执行以下命令，然后再浏览器输入localhost:4000查看。

$ hexo generate
$ hexo server

3.3 配置Hexo
打开Hexo根目录下的_config.yml，修改以下代码：

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:username/username.github.io.git
  branch: master

为了能够使Hexo部署到GitHub上，需要安装一个插件：

$ npm install hexo-deployer-git --save

然后，执行下列指令即可完成部署：

$ hexo generate
$ hexo deploy

之后，可以通过在浏览器键入：username.github.io进行浏览。
你已成功完成博客的创建，现在头等大事就是编写属于你自己的文章了，现在开始动手吧。
